Flujos de trabajo típicos de Prisma

Como se mencionó anteriormente, hay dos formas de "obtener" su modelo de datos en el esquema Prisma. Según el enfoque que elija, su flujo de trabajo principal de Prisma puede verse diferente.

Migraciones SQL e introspección

El flujo de trabajo típico cuando se usan migraciones SQL e introspección se ve de la siguiente manera:

  1. Ajuste manualmente el esquema de su base de datos usando SQL
  2. (Re-)inspeccione su base de datos
  3. Opcionalmente (re-)configure su API de cliente Prisma
  4. (Re-)generar Prisma Client
  5. Use Prisma Client en su código de aplicación para acceder a su base de datos

Prisma Migrate

Con Prisma Migrate, el flujo de trabajo se ve ligeramente diferente:

  1. Ajuste manualmente su modelo de datos Prisma
  2. Migre su base de datos utilizando los comandos prisma migrate de la CLI
  3. (Re-)generar Prisma Client
  4. Use Prisma Client en su código de aplicación para acceder a su base de datos
